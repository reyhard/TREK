## Migration Rollback Classification

Source: `server/src/db/migrations.ts` (combined fork + upstream migration array)

Migrations are numbered sequentially from 1 (`migrations.length` total). Classified by structural impact:

### Backward-Compatible (194 migrations)
Previous fork build can run on the migrated database without errors:

- All `ALTER TABLE ADD COLUMN` with NULL/default — old code tolerates unknown columns
- All `CREATE TABLE IF NOT EXISTS` / `CREATE INDEX IF NOT EXISTS` — new tables the old code never queries
- All `INSERT OR IGNORE INTO addons` — new addon registry rows
- All `UPDATE` with data corrections (trim whitespace, renumber sort_order, clear stale cache)
- Migration range: ~1-20 (ALTER TABLE boilerplate), 21+ (CREATE TABLE for new features)

### Forward-Only Non-Destructive (approx. 6-8 migrations)
Rollback requires restoring a pre-deployment backup; the old fork image cannot read the migrated schema:

| Migration | Operation | Data Impact |
|-----------|-----------|-------------|
| Budget items NOT NULL DEFAULT 1 → nullable | DROP + recreate budget_items | Existing data migrated to new schema |
| trip_photos schema normalization (×2) | DROP + recreate trip_photos | Data migrated via INSERT INTO ... SELECT |
| journey_photos schema evolution (×3) | DROP + recreate journey_photos | Data backfilled into new schema |
| oauth_clients nullable user_id | DROP + recreate oauth_clients | Data preserved, FK relaxed |
| day_accommodations NOT NULL place_id → nullable | DROP + recreate day_accommodations | ON DELETE CASCADE → SET NULL |
| idempotency_keys wider PK | DROP + recreate idempotency_keys | Data preserved, PK widened |
| Journey gallery refactor (migration 121) | DROP journey_photos_old, DELETE FROM journey_entries | Old data migrated to new gallery model |
| Photos provider abstraction layer | DROP + recreate trip_photos + journey_photos | Photos migrated to trek_photos registry |
| OAuth NULL-audience token cleanup | DELETE FROM oauth_tokens WHERE audience IS NULL | Tokens revoked (re-issuable) |
| Stale Google photo URL cleanup | UPDATE places SET image_url = NULL | Re-fetchable |
| place_regions cache clear | DELETE FROM place_regions | Re-derivable from Nominatim |

### Destructive
**NONE** — all DROP TABLE operations are preceded by data migration to a new table. No operation is prohibited without separate approval.

## Intentional Behavior Changes

1. **Public transit references in geo:read scope** — labels/descriptions updated across all 23 locales
2. **Settings normalization** — `normalizeSettings()` handles old string `dark_mode`, unknown temperature/distance units, security-sensitive defaults
3. **Safe metadata parsing** — `safeParseMetadata()` replaces raw `JSON.parse` in 7 production components
4. **Generic transit fields** — `status` and `confirmation_number` now editable in TransitJourneyModal
5. **Multi-day date display** — end date shown when different from start date in transport detail modal
6. **Focus restoration** — modals save/restore focus for accessibility
7. **Accessibility utilities** — `respectReducedMotion()`, `isRtlLanguage()`, keyboard handlers
8. **Movement stats and live events** — derived movement totals, accommodation event debouncing, Dexie write-through
9. **Reposition controls** — drag-to-reposition with blue glow, click suppression
10. **Deduplicated visible connections** — IDs deduplicated on parse and toggle
11. **Booking route endpoints in bounds** — reservation endpoints included in map fitBounds
12. **Scope description corrections** — `places:read` no longer mentions transit stops
